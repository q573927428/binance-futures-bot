# 增量同步修复说明

## 问题概述

原增量同步实现存在以下关键问题：

### ❌ 问题1：未收盘K线被过滤掉
```typescript
const newKlineData = klineData.filter(item => item.timestamp > lastTimestamp)
```
**问题本质**：Binance返回的最后一根K线可能是未收盘的，时间戳与本地相同。但上述过滤条件 `item.timestamp > lastTimestamp` 会排除时间戳相等的情况，导致未收盘K线永远不会更新。

### ❌ 问题2：startTime +1 破坏K线对齐
```typescript
const startTime = (lastTimestamp + 1) * 1000
```
**问题本质**：K线是严格按周期对齐的（如10:00:00、11:00:00）。使用 `+1` 会导致 `10:00:01`，可能跳过整点K线或导致Binance返回异常范围。

### ❌ 问题3：缺少update机制
原系统只有 `appendSimpleKLineData` 函数，没有专门更新最后一根K线的机制。

## 修复方案

### ✅ 修复1：新增 updateLastKLine 函数
在 `server/utils/kline-simple-storage.ts` 中添加：
```typescript
export function updateLastKLine(
  symbol: string,
  timeframe: KLineTimeframe,
  bar: KLineData
): boolean
```
功能：专门用于更新最后一根未收盘K线。

### ✅ 修复2：修改增量同步逻辑
在 `server/modules/kline-simple-sync/index.ts` 的 `syncSymbolKLine` 方法中：

**旧逻辑**：
```typescript
const startTime = (lastTimestamp + 1) * 1000
const newKlineData = klineData.filter(item => item.timestamp > lastTimestamp)
appendSimpleKLineData(symbol, timeframe, newKlineData)
```

**新逻辑**：
```typescript
const startTime = lastTimestamp * 1000  // 不加1

// 分离数据
let newBars: KLineData[] = []
let updateBar: KLineData | null = null

for (const item of klineData) {
  if (item.timestamp > lastTimestamp) {
    newBars.push(item)  // 新K线
  } else if (item.timestamp === lastTimestamp) {
    updateBar = item    // 未收盘K线
  }
}

// 1. 更新最后一根K线
if (updateBar) {
  updateLastKLine(symbol, timeframe, updateBar)
}

// 2. 追加新K线
if (newBars.length > 0) {
  appendSimpleKLineData(symbol, timeframe, newBars)
}
```

## 核心改进

### 1. 正确处理未收盘K线
- **旧逻辑**：未收盘K线被过滤掉，永远不更新
- **新逻辑**：未收盘K线被识别并更新

### 2. 保持K线对齐
- **旧逻辑**：`startTime = lastTimestamp + 1` 可能破坏对齐
- **新逻辑**：`startTime = lastTimestamp` 保持严格对齐

### 3. 支持两种操作
- **append**：已收盘K线（时间戳 > lastTimestamp）
- **update**：未收盘K线（时间戳 === lastTimestamp）
