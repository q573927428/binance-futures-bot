# 新的K线数据存储方案设计

## 设计目标
1. 简化存储结构：不再按年份分割文件，每个交易对每个周期一个文件
2. 限制文件大小：每个文件最多保存22000条K线数据
3. 提高读取性能：前端可以直接读取单个文件的所有数据
4. 保持向后兼容：逐步迁移现有数据

## 新的文件结构
```
data/kline/
├── BTCUSDT/
│   ├── 1h.json          # 1小时K线数据（最多22000条）
│   ├── 15m.json         # 15分钟K线数据
│   ├── 4h.json          # 4小时K线数据
│   ├── 1d.json          # 日K线数据
│   └── 1w.json          # 周K线数据
├── ETHUSDT/
│   ├── 1h.json
│   └── 15m.json
└── latest-cache/        # 最新数据缓存（可选）
    ├── BTCUSDT-1h.json
    └── ETHUSDT-1h.json
```

## 新的数据文件格式

```json
{
  "symbol": "BTCUSDT",

```
